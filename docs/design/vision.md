# Vision — Dioptron

## The dual purpose

Dioptron serves two audiences through the same capability surface:

**For the operator:** A sovereign daily-driver browser replacement. Browse, research, ingest, and act on the web without surveillance, fingerprinting, or dependency on any cloud service. Full control over identity, privacy, and active capabilities. The web on your terms.

**For nous agents:** Structured, integrated web access as a first-class capability. Today, agents interact with the web through crude tool-use (fetch a URL, parse HTML). Dioptron gives agents the same rich, session-aware, knowledge-integrated web interaction the operator has — browsing, ingesting into the shared knowledge graph, following links, understanding context, acting with full audit and cost accounting.

## The ecosystem topology

Every forkwright project participates in a topology where each system makes the others more capable:

- **Aletheia** provides the cognitive substrate — agents, memory, reasoning, knowledge graph
- **Dioptron** provides the web surface — the instrument through which both operator and agents see through the web
- **Akroasis** provides the signal environment — network defense, RF intelligence, posture awareness
- **Harmonia** provides the media surface — organized, quality-managed media with audiophile playback
- **Thumos** provides the hardware surface — sovereign mobile with identity protection and radio intelligence
- **Hamma** provides the transport layer — mesh networking that connects all surfaces across devices
- **Kanon** provides the standards and automation — the quality floor everything builds on
- **Gnomon** provides the architecture theory — how intelligence emerges from topology

Dioptron is the missing piece that connects aletheia's agents to the open web. Without it, nous agents are blind to the internet — they can make API calls but they can't browse, can't understand pages in context, can't build knowledge from web research, can't act on web surfaces. With dioptron integrated, a nous agent can:

- Research a topic by browsing, reading, and ingesting — building its knowledge graph from web sources
- Monitor web pages for changes (standing queries) and surface findings to the operator
- Fill forms, submit requests, manage accounts — with the operator's grants and full audit
- Analyze web security posture — TLS, supply chain, certificate transparency — as part of akroasis-integrated defense
- Generate training data from every web interaction — improving its own future performance

The peer tenancy model (R2) is the architectural commitment that makes this real. The agent doesn't get a "simplified agent API" — it gets the same trait surface the operator uses. Same capabilities, same observability, same cost accounting. Differences are grants, not capabilities.

## What this is not

- Not a browser with AI bolted on (the AI is a peer tenant, not a feature)
- Not a proxy (it renders, executes, and understands content natively)
- Not a scraper (it builds persistent, tiered, searchable knowledge)
- Not a security tool with a browser attached (security is one of five co-equal purposes)
